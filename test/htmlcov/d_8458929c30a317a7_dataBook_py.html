3:mod:`cape.pyover.dataBook`: pyOver data book module  

4===================================================== 

5 

6This module contains functions for reading and processing forces, 

7moments, and other statistics from cases in a trajectory. Data books are 

8usually created by using the :func:`cape.pyover.cntl.Cntl.ReadDataBook` 

9function. 

10 

11 .. code-block:: python 

12  

13 # Read OVERFLOW control instance 

14 cntl = pyOver.Cntl("pyOver.json") 

15 # Read the data book 

16 cntl.ReadDataBook() 

17 # Get a handle 

18 DB = cntl.DataBook 

19  

20 # Read a line load component 

21 DB.ReadLineLoad("CORE_LL") 

22 DBL = DB.LineLoads["CORE_LL"] 

23 # Read a target 

24 DB.ReadTarget("t97") 

25 DBT = DB.Targets["t97"] 

26  

27Data books can be created without an overall control structure, but it 

28requires creating a run matrix object using 

29:class:`cape.pyover.runmatrix.RunMatrix`, so it is a more involved 

30process. 

31 

32Data book modules are also invoked during update and reporting 

33command-line calls. 

34 

35 .. code-block:: console 

36  

37 $ pyfun --aero 

38 $ pyfun --ll 

39 $ pyfun --triqfm 

40 $ pyfun --report 

41 

42The available components mirror those described on the template data 

43book modules, :mod:`cape.cfdx.dataBook`, :mod:`cape.cfdx.lineLoad`, and 

44:mod:`cape.cfdx.pointSensor`. However, some data book types may not be 

45implemented for all CFD solvers. 

46 

47:See Also: 

48 * :mod:`cape.cfdx.dataBook` 

49 * :mod:`cape.cfdx.lineLoad` 

50 * :mod:`cape.cfdx.pointSensor`

113def ReadFomocoComps(fname): 

114 """Get list of components in an OVERFLOW fomoco file

116 :Call:

118 :Inputs:

123 List of components 

124 :Versions: 

125 * 2016-02-03 ``@ddalle``: First version 

126 """ 

127 # Initialize components 

128 comps = [] 

129 # Open the file 

130 f = open(fname, 'r') 

131 # Read the first line and first component. 

132 comp = f.readline().strip() 

133 # Loop until a component repeats 

134 while comp not in comps:

144 # Close the file 

145 f.close()
